1.What is DOM? What is the purpose of DOM?

Web documents have a programming interface called the Document Object Model (DOM). Since that software can alter the document's structure, appearance, and content, it serves as a representation of the page. Programming languages can communicate with a page by interacting with the nodes and objects that the DOM uses to represent the document.

An application programming interface (API) for HTML and XML documents is called the Document Object Model (DOM). It specifies how a document is accessed and handled, as well as its logical structure.

3.What is event bubble?

The DOM has a concept known as event bubbling (Document Object Model). The event bubbles up (or you could say is transmitted or propagated) to the element's parent and grandparent elements in the DOM tree until it reaches the root element once an element receives an event.


4.What is callback function? why will you use it?

A callback function is a function that receives an argument from another function and is subsequently called from inside the outer function to finish a task or activity.

Due to the fact that many JavaScript activities are asynchronous, callback methods are required. Instead, it runs in the background when the rest of the code is being executed. The function of a callback is to run code in response to an event.
